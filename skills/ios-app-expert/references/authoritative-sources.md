# Authoritative Sources for iOS Verification

## Use Sources in This Order

1. Apple Developer Documentation for API behavior and platform capabilities.
2. Apple release notes for iOS, Xcode, and SDK-specific behavior changes.
3. App Store Review Guidelines and Apple policy pages for submission requirements.
4. WWDC session pages and official Apple sample code for recommended patterns.
5. Official third-party SDK documentation and release notes.
6. Community sources only to form hypotheses, never as final authority.

## Verification Checklist

- Confirm publish date and last-updated date.
- Confirm alignment with target iOS and Xcode versions.
- Confirm whether a claim is documented behavior or anecdotal behavior.
- Cross-check critical compatibility or policy claims across multiple primary sources.
- Capture URLs for every externally verified statement.
